# Telco Churn Database Project

## Описание проекта

Проект посвящён разработке реляционной базы данных для анализа оттока клиентов телекоммуникационной компании.

Цель проекта — реализовать полный цикл работы SQL-разработчика:
- проектирование структуры БД по ER-диаграмме;
- настройка связей между таблицами;
- наполнение базы большим объёмом данных;
- оптимизация производительности аналитических запросов;
- валидация корректности данных.

База предназначена для последующего выполнения аналитических SQL-запросов по клиентской активности и оттоку.

---

## Структура базы данных

В проекте реализованы следующие таблицы:

### Основные таблицы
- **Customers** — профиль клиента и финансовые показатели;
- **ServiceDetails** — детализация подключённых услуг.

### Справочники
- **Contracts** — типы контрактов;
- **PaymentMethods** — способы оплаты;
- **InternetServices** — интернет-услуги.

---

## Связи таблиц

Customers (1:1) → ServiceDetails
Customers (N:1) → Contracts
Customers (N:1) → PaymentMethods
ServiceDetails (N:1) → InternetServices


Связи реализованы через внешние ключи с каскадным обновлением и удалением.

---

## Генерация и загрузка данных

Для наполнения базы был разработан Python-скрипт с использованием библиотек:

- **Faker** — генерация реалистичных персональных и финансовых данных;
- **random** — распределение категорий;
- **pandas** — формирование CSV-файлов.

Сгенерировано:
- **1000 строк в таблице Customers;**
- **1000 строк в таблице ServiceDetails.**

Импорт CSV выполнен через команду:

LOAD DATA LOCAL INFILE

Данный способ обеспечивает быструю массовую загрузку данных и минимизирует нагрузку на сервер.
